## 1. 계층 구조 개요

```
[ 난이도 ]                  ← 가장 큰 틀. 선택 후 엔딩 전까지 고정.
    │
    └─ [ 런 (Run) ]         ← 패배 시 리셋. 기록칩은 런 안에서 계속 누적.
           │
           └─ [ Day ]       ← 매일 반복. 스토리 전개 단위.
                  │
                  └─ [ 탐사 / 라운드 ]   ← 하루 안의 전투·이벤트.
```

- 패배(전멸) → 런 리셋. **난이도는 유지.**
- 엔딩 → **기록칩 포함 전체 초기화.** 난이도 선택부터 다시.
- 게임을 껐다 켜도 진행 중인 런이 유지됨.

## 2. 난이도 목록

| id | 표시 명칭 | 설명 | 비고 |
|----|-----------|------|------|
| `normal` | **부식기** | 기본 규칙. 탐사 실패 = 영구사망. | 정규 난이도 |
| `hard` | **심해류** | 매 Day 부식 가속. 영입 비용 상승. 기록칩 보정 없음. | 클리어 후 해금 예정 |
| `easy` | **부식 전** | 기록칩 보정 적극 적용. 탐사 실패 시 전멸 없이 부상만. | 추후 검토 |

> 난이도 명칭은 세계관 용어 기반. 확정 전 변경 가능.

## 3. 저장 키 구조

| localStorage 키 | 수명 | 내용 |
|-----------------|------|------|
| `nr_difficulty` | 엔딩 후 초기화 | 선택한 난이도 id |
| `nr_run` | 런 패배 시 초기화 | 현재 런 정보 (런 번호, Day 등) |
| `neural_rust_save` | 런 패배 시 초기화 | arc, day 등 인게임 수치 |
| `neural_rust_story` | 런 패배 시 초기화 | 스토리 플래그, 진행도 |
| `nr_characters` | 런 패배 시 초기화 | 캐릭터 목록 |
| `nr_record_chips` | 엔딩 후만 초기화 | 기록칩 (런 패배로는 절대 지워지지 않음) |
| `neural_rust_settings` | 수동만 | 설정 |
| `neural_rust_keybinds` | 수동만 | 키 바인딩 |

## 4. nr_difficulty 데이터 구조

```js
// localStorage 키: 'nr_difficulty'
{
  id:    'normal',    // 'easy' | 'normal' | 'hard'
  label: '부식기',    // 표시명
}
```

---

## 5. nr_run 데이터 구조

```js
// localStorage 키: 'nr_run'
{
  runCount:  3,             // 이번 난이도에서 몇 번째 런인지 (패배마다 +1)
  startedAt: 1710000000000, // 이 런의 시작 타임스탬프
  day:        14,           // 현재 Day (neural_rust_save의 day와 동기화)
}
```

> `runCount`는 난이도 선택 시 1로 초기화, 런 패배마다 +1.
> 로비 화면에서 "3번째 도전 중" 표시에 활용.

## 6. SaveManager에 추가할 메서드 (구현 예정)

```js
// 난이도 선택 + 첫 런 시작
SaveManager.startNewGame(difficultyId)
  // nr_difficulty 저장
  // nr_run = { runCount: 1, startedAt: now, day: 1 }
  // neural_rust_save, story, nr_characters 초기화
  // nr_record_chips는 건드리지 않음

// 런 패배 처리
SaveManager.resetRun()
  // nr_run.runCount++
  // nr_run.startedAt = now, day = 1
  // neural_rust_save, story, nr_characters 초기화
  // nr_difficulty, nr_record_chips는 건드리지 않음

// 엔딩 처리
SaveManager.clearEnding()
  // nr_difficulty 삭제
  // nr_run 삭제
  // neural_rust_save, story, nr_characters 초기화
  // nr_record_chips 초기화 (엔딩 후만 허용)

// 로비 상태 판별
SaveManager.getLobbyState()
  // 반환: 'fresh' | 'playing' | 'ended'
  // 'fresh'   — nr_difficulty 없음 → 새 게임 시작 가능
  // 'playing' — nr_difficulty + nr_run 있음 → 이어하기 가능
  // 'ended'   — 엔딩 플래그 있음 → 엔딩 화면 표시
```

---

## 7. 로비 분기 흐름

```
LobbyScene 진입
    │
    ├─ getLobbyState() === 'fresh'
    │      → [ 시작하기 ] → 난이도 선택 → 런 시작
    │
    ├─ getLobbyState() === 'playing'
    │      → [ 이어하기 ]            → AtelierScene
    │      → [ 이번 런 포기하기 ]    → 확인 팝업 → resetRun() → AtelierScene
    │
    └─ getLobbyState() === 'ended'
           → 엔딩 결과 표시
           → [ 처음부터 시작하기 ]  → clearEnding() → 난이도 선택
```
