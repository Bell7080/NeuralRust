# IRONWAVE - 전투 기획 문서 v0.1

> 📌 **문서 연동 안내 (개발자/AI 참고용)**
> 이 문서는 전투 시스템 전용 기획 문서입니다.
> 기존에 존재하는 캐릭터, 스킬, 탐사, 배치 관련 문서와 연동하여 개발합니다.
> 각 섹션 상단에 연동 대상 파일/문서를 주석으로 명시했습니다.
> 실제 개발 시 파일을 하나씩 제공할 예정이며, 제공된 파일 기준으로 연동 작업을 진행합니다.

## 1. 전투 진입 구조

### 1-1. 탐사 난이도 슬롯
<!-- 
연동: 탐사탭 문서 / ExpeditionManager
- 탐사 진입 전 슬롯머신을 돌려 난이도 3개 중 하나를 선택
- 난이도는 코그 등급(1~10) 기반으로 결정
- 선택한 난이도가 해당 탐사 전체 코그 상한이 됨
- 파티 구성 가능한 캐릭터는 코그 상한 이하로 제한
-->

- 탐사 진입 시 슬롯머신으로 난이도 **3개 중 1개 선택**
- 난이도는 **코그 등급(1~10)** 으로 표기
- 선택한 난이도 = 해당 탐사의 **코그 상한**
- 코그 상한 초과 캐릭터는 파티 편성 불가
- 난이도가 높을수록 적 스탯풀 상승, 보상 증가

### 1-2. 파티 구성 및 배치
<!--
연동: 탐사탭 문서 / 배치탭 문서 / CharacterManager
- 코그 상한 내에서 보유 캐릭터로 파티 구성
- 3x3 배치탭에서 5x5 타일맵 내부(캐릭터 배치 가능 구역)에 배치
- 라운드 사이에 재배치 가능
- 잠수정은 5x5 중앙 칸(고정)
-->

- 코그 상한 내에서 보유 캐릭터로 파티 구성
- **5x5 타일맵** 내부에 캐릭터 배치 (한 칸당 최대 3명)
- 잠수정은 **중앙 칸 고정**
- 라운드 사이마다 **재배치 1회** 가능

```
[ 외곽 레일 - 24칸 ]
X  X  X  X  X  X  X
X  ㅁ  ㅁ  ㅁ  ㅁ  ㅁ  X
X  ㅁ  ㅁ  ㅁ  ㅁ  ㅁ  X
X  ㅁ  ㅁ  ㅇ  ㅁ  ㅁ  X   ← ㅇ = 잠수정(중앙 고정)
X  ㅁ  ㅁ  ㅁ  ㅁ  ㅁ  X
X  ㅁ  ㅁ  ㅁ  ㅁ  ㅁ  X
X  X  X  X  X  X  X

X = 외곽 레일 (적 입구 후보)
ㅁ = 캐릭터 배치 가능 칸
ㅇ = 잠수정
```

## 2. 라운드 구조

### 2-1. 전체 흐름
<!--
연동: RoundManager / SlotMachineManager
- 탐사 1회 = 최대 5라운드
- 매 라운드 시작마다 라운드 슬롯 실행
- 라운드 결과에 따라 전투 / 이벤트 / 상점 중 하나 진행
-->

```
[탐사 시작]
    ↓
[탐사 난이도 슬롯] - 3개 중 1개 선택

[파티 구성]
    ↓
[라운드 슬롯] ──────────────────────────────────────────┐
    ↓                                                   │
[전투 / 이벤트 / 상점]                                   │
    ↓                                                   │
[일시 결산]                                              │
    ↓                                                   │
[재배치]                                                 │
    ↓                                                   │
[다음 라운드] ───────────────────────────────────────────┘
    ↓ (5라운드 완료 또는 철수)
[최종 결산]
    ↓
[Day 종료 → 다음 날 경영]
```

### 2-2. 라운드 슬롯
<!--
연동: SlotMachineManager / RoundManager
- 슬롯 결과 목록: 전투 / 낚시 이벤트 / 잔해수집 이벤트 / 전선연결 이벤트 / 심해상점 / 표류체 발견 / 추적 신호 / 긴급 정비 / 철수
- 철수 슬롯은 위기 조건 충족 시 강제 포함
- 위기 조건: 파티 캐릭터 절반 이상 전투불능 OR 잠수정 체력 50% 이하
- 위기 조건 충족 후 미선택 시 랜덤 결과로 대체
-->

**라운드 슬롯 결과 목록**

| 슬롯 결과 | 설명 |
|-----------|------|
| 전투 | 이형과 전투 진행 |
| 낚시 이벤트 | 낚시꾼 활용 자원 수집 |
| 잔해 수집 이벤트 | 잠수부 활용 고철/아이템 수집 |
| 전선 연결 이벤트 | AI 활용 화폐 수집 |
| 심해 상점 | 심해화폐로 아이템 구매 |
| 표류체 발견 | 즉발형 도박 이벤트 (자원 or 함정) |
| 추적 신호 | 선택형 도박 이벤트 (고보상 or 기습전투) |
| 긴급 정비 | 잠수정 체력 소량 회복 (낮은 확률) |
| 철수 | 탐사 즉시 종료 후 결산 |

**철수 슬롯 강제 포함 조건**
- 파티 캐릭터 절반 이상 전투불능
- 잠수정 체력 50% 이하
- 위 조건 충족 시 다음 라운드 슬롯 3개 중 1개는 반드시 철수
- 철수를 선택하지 않으면 나머지 결과 중 랜덤 적용

<!--
연동: RailSlotManager / BattleManager

- 정지된 위치에서 랜덤으로 입구 1~4개 활성화

- 외곽 **24칸 레일**을 슬롯머신처럼 회전 후 정지
- 정지 위치 기준으로 입구 **1~4개** 랜덤 활성화
- 전투 진입 전 **적 정보 공개** (이형 종류, 수, 입구 위치)
- 플레이어는 적 정보 확인 후 배치 조정

### 3-2. 전투 진행
<!--
연동: BattleManager / CharacterManager / EnemyManager
- 실시간 자동전투
- 캐릭터는 고정 배치, 포지션 기반 사거리로 자동 공격
- 이형은 외곽 레일 입구에서 등장, 칸을 걸어서 이동
- 잠수정 파괴 또는 전원 전투불능 시 탐사 실패 (전원 영구 사망)
-->

- **실시간 자동전투**
- 캐릭터는 배치 칸에 **고정**, 포지션 기반 사거리로 자동 공격
- 이형은 입구에서 등장, **칸 단위로 걸어서 이동**
- 잠수정이 파괴되거나 전원 전투불능 시 **탐사 실패**
- 탐사 실패 = 파티 전원 **영구 사망**

**전투 중 지원 슬롯**
- 전투 중 **1회** 지원 슬롯 사용 가능
- 슬롯 결과 3개 중 1개 선택 (예: 체력 회복 / 특정 이형 즉시 제거 / 이번 라운드 공격력 증가)

### 3-4. 캐릭터 사망 및 회수
<!--
연동: CharacterManager / BattleManager
- 전투 중 체력 0 = 전투불능 상태로 해당 칸에 잔류
- 같은 라운드 내 전투 종료(잠수정 생존) 시 회수 가능
- 회수된 캐릭터는 탐사 내 전투 불가

|------|------|------|

| 영구 사망 | 탐사 실패 | 캐릭터 데이터 삭제 |

## 4. 스탯 시스템

### 4-1. 플레이어 캐릭터 스탯
<!--
연동: CharacterManager / StatManager
- 스탯 필드명은 기존 캐릭터 데이터 파일 기준으로 연동
- 코그 등급은 캐릭터 생성 시 확정, 이후 고정 (아이템으로만 보정 가능)
- 건강 스탯 = 라운드/탐사 종료 시 회복되는 체력량
-->

| 스탯 | 설명 |
|------|------|
| 체력 | HP (피통) |
| 건강 | 라운드/탐사 종료 시 회복되는 체력량 |
| 공격 | 공격력 |

**코그 등급**
- 캐릭터 생성 시 확정, 이후 **고정**
- 아이템 장착으로 **보정 가능** (등급 증감)
- 등급별 스탯 범위 (예시, 조정 가능)

| 코그 등급 | 스탯 합산 범위 |
|-----------|---------------|
| 1 | 15 ~ 20 |
| 2 | 21 ~ 40 |
| 3 | 41 ~ 60 |
| ... | ... |
| 10 | 네임드 캐릭터 전용 |

## 6. 스킬 시스템

### 6-1. 스킬 충전 방식
<!--
연동: SkillManager / 스킬 데이터 파일
- 스킬 데이터 파일에 충전 타입 필드 추가 필요
- 충전 타입: ATTACK_COUNT(공격 횟수) / HIT_COUNT(피격 횟수) / TIME(시간)
- 각 스킬마다 충전 타입과 충전 수치를 스킬 데이터 파일에서 직접 정의
-->

스킬 충전 방식은 **스킬별로 독립 정의**

| 충전 타입 | 설명 | 유리한 캐릭터 |
|-----------|------|---------------|
| ATTACK_COUNT | 공격 N회마다 발동 | 민첩 높은 캐릭터 |
| HIT_COUNT | 피격 N회마다 발동 | 잠수부·AI (받는 피해 높음) |
| TIME | N초마다 발동 | 공속 무관, 안정적 |

> 스킬 데이터 파일에서 각 스킬의 충전 타입과 수치를 커스텀

## 8. 이벤트 시스템

### 8-1. 수집 이벤트 구조
<!--
연동: EventManager / TileMapManager / CharacterManager
- 이벤트 라운드에서도 5x5 타일맵 동일하게 사용
- 자원 포인트가 타일 위에 숨겨진 채로 시작
- 플레이어가 캐릭터를 배치 후 완료 시 각 칸 결과 공개
- 직업 일치 시 100%, 불일치 시 25% 수집률
-->

수집 이벤트는 **5x5 타일맵** 동일하게 사용

**진행 방식**
1. 타일에 자원 포인트 숨겨진 채로 시작
2. 플레이어가 캐릭터를 원하는 칸에 배치
3. 완료 버튼 클릭 시 각 칸 자원 공개 + 수집 결과 확인

**수집률**

| 배치 캐릭터 | 수집률 |
|-------------|--------|
| 해당 직업 캐릭터 | 100% |
| 다른 직업 캐릭터 | 25% |
