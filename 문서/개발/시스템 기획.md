| 항목 | 결정 |

## 2. 장르 정의

**스토리 중심 관리 시뮬레이션 + 디펜스**

- 로보토미 코퍼레이션을 레퍼런스로 삼되, 게임성 비중은 낮추고 서사 비중을 높인다
- 직접 조작하는 액션 요소는 최소화. 플레이어는 **배치와 판단**으로 게임을 진행
- 매일 반복되는 운영 루틴이 플레이어를 이 세계에 살게 만들고, 그 루틴 사이에 스토리가 끼어든다
- 로그라이크 요소는 현 단계에서 보류

## 3. 핵심 루프

```
[ 아침 — 경영 페이즈 ]
  스토리 씬 / NPC 대화
  ↓
  공방(Atelier) 에서 편성
  └─ AI 전류 채굴 라인 배치
  └─ 낚시꾼 보초 배치
  └─ 다이버 잠수 목표 설정
  └─ 엔지니어 외주 협상 (필요 시)

[ 점심 — 운영 페이즈 ]
  탐색 결과 이벤트 처리
  └─ 괴수 방어 (낚시꾼 디펜스)
  └─ 잠수 결과 수령 (고철 · 로어 · 기타)
  └─ AI 채굴 Arc 수령

[ 저녁 — 결산 페이즈 ]
  Arc · 자원 정산
  스토리 씬 / 캐릭터 대화

[ 다음날 아침 ]
  전날 이벤트 후일담 / 새 대화 이벤트
  → 아침 경영 페이즈로 순환
```

## 4. 공방 (Atelier) UI 구조

플레이어는 인게임 진입 시 공방 화면에 들어온다.

### 화면 레이아웃

```
┌──────────────────────────────────────────────────────────────┐
│  [창고] [도감] [회상]      DAY X  |  XXX Arc      [설정]    │
│ ─────────────────────────────────────────────────────────── │
│                                                              │
│                                                              │
│                    [ 탭 패널 영역 ]                           │
│                                                              │
│                                                              │
│  ─────────────────────────────────────────────────────────  │
│          [영입] [탐사대] [시설] [외주] [드레지]               │
│                                                              │
│                       [ 탐  색 ]                             │
└──────────────────────────────────────────────────────────────┘
```

### 탭 목록

| 탭 | 위치 | 역할 |
|----|------|------|
| 탐색 | 하단 중앙 (기본 탭) | 탐사 구역 선택 및 출발 |
| 영입 | 우측 상단 | 낚시꾼 / 다이버 고용 |
| 탐사대 | 우측 상단 | 탐색에 나갈 팀 편성 |
| 시설 | 우측 상단 | 공방 내실 강화 (탐사대 인원 증가 등) |
| 외주 | 우측 상단 | 엔지니어에게 하드웨어 제작 의뢰 |
| 드레지 | 우측 상단 | AI를 하드웨어에 호출/삽입 |
| 창고 | 좌측 상단 | 고철·기록물 등 수집 아이템 관리 |
| 도감 | 좌측 상단 | 캐릭터 특성·용어 사전 |
| 회상 | 좌측 상단 | 지난 스토리 씬 다시 보기 |

### 파일 구조

```
Games/Codes/Scenes/Atelier/
  AtelierScene.js   — 메인 씬 (탭 라우팅, 설정 버튼)
  AtelierHUD.js     — 상단 Day / Arc 표시
  AtelierTabs.js    — 탭 버튼 공통 빌더 (makeAtelierTab)
  tabs/
    Tab_Explore.js  — 탐색 탭
    Tab_Stubs.js    — 미구현 탭 플레이스홀더 전체
```

## 5. 자원 구조

### Arc (아크) — 핵심 화폐

- 바다에 매립된 전선에서 흘러나오는 전류를 정제한 에너지 단위
- 거대한 톱니바퀴 선박을 바다 위에 띄우는 데 필수 동력원이자 이 세계의 화폐
- AI를 전류 채굴 라인에 배치하면 매일 일정량의 Arc를 수입으로 획득
- 모든 고용 · 외주 · 장비 구매는 Arc로 결제

| 지출 항목 | 내용 |
|-----------|------|
| 낚시꾼 고용 | 일당 Arc 지급 |
| 다이버 고용 | 잠수 의뢰 Arc 지급 |
| 엔지니어 외주 | 하드웨어 제작 의뢰 Arc + 고철 |
| 몸체 수리 | 부식된 AI 하드웨어 유지보수 |
| 세력 소속 인력 고용 | 일반 고용보다 높은 Arc 필요 |

### 고철 (Scrap)

- 다이버가 심해에서 수집하는 붕괴 이전 문명의 금속 잔해
- 엔지니어에게 Arc와 함께 지급해 AI 하드웨어(몸체) 제작에 사용
- 고철의 시대와 품질에 따라 제작 몸체 성능이 달라짐
- 소금바다 증기에 의해 몸체가 서서히 부식(Rust)되므로 수리용으로도 소비

### 기타 수집물

- 로어(기록물), 출처 불명 교역품, 세력 해금 트리거가 되는 특수 유물 등
- 각 물품 활용 방식은 추후 설계

## 6. 직군 구조

### AI (드레지 대상)

| 항목 | 내용 |
|------|------|
| 핵심 역할 | Arc 채굴. 전선에서 전류를 획득해 Arc 생산 |
| 수익 구조 | 배치된 AI 수 × 개체 성능 = 일일 Arc 수입 |
| 스토리 역할 | 드레지(Dredge)를 통해 탄생한 각 AI가 독립적인 스토리 진행 |
| 리스크 | 몸체 부식. 방치 시 능력치 저하 → 기능 장애 → 야생화 |
| 획득 방법 | 엔지니어에게 몸체 제작 의뢰 → 드레지(Dredge) 수행 → AI 탑재 완료 |

**드레지(Dredge)**: 소프트웨어 상태로 전선을 부유하는 AI를 고철 몸체에 불러오는 행위.

### 낚시꾼 (Fisher)

| 항목 | 내용 |
|------|------|
| 역할 | 톱니바퀴 선박의 외곽을 지키는 보초. 괴수 및 외부 위협 방어 |
| 게임적 기능 | 디펜스 라인 구성. 배치와 장비로 방어력 결정 |
| 고용 구조 | 초기: 일반 고용 / 이후: 세력 소속 낚시꾼 단체 계약 가능 |
| 리스크 | 부상 / 사망 / 정신적 소모 |

### 다이버 (Diver)

| 항목 | 내용 |
|------|------|
| 역할 | 심해로 내려가 고철 · 로어 · 기타 물품 수집 |
| 게임적 기능 | 고철 공급 + 로어 해금 + 특수 이벤트 트리거 |
| 고용 구조 | 초기: 일반 고용 / 이후: 세력 소속 잠수사 집단 외주 가능 |
| 리스크 | 잠수 깊이에 따라 위험도 상승. 수집물 품질도 달라짐 |

### 엔지니어 (Engineer)

| 항목 | 내용 |
|------|------|
| 역할 | AI 하드웨어(몸체) 제작 전문가 |
| 고용 구조 | 항상 외주. Arc + 고철을 지급하고 협상으로 제작 의뢰 |
| 발전 구조 | 초기: 일반 엔지니어 / 이후: 세력 소속 엔지니어 집단에 접근 가능 |
| 특이점 | 세력마다 엔지니어 작풍이 달라 몸체 특성이 차별화됨 |

## 7. Cog (코그) — 위험도 등급 시스템

Cog는 이 세계의 **위험 등급 분류 체계**. 탐색 구역, AI, 괴수, 기록물 등에 모두 적용되는 공통 척도.

> 세력을 칭하는 용어: **마스트(Mast)**. 각 마스트의 수장은 **앵커(Anchor)**.

### Cog 등급 체계 (안)

| 등급 | 명칭 | 기준 |
|------|------|------|
| Cog 1 | — | 일반적인 위험. 낚시꾼 기본 장비로 대응 가능 |
| Cog 2 | — | 중급 위험. 숙련된 인력 필요 |
| Cog 3 | — | 고위험. 전문 장비 및 다수 인원 필요 |
| Cog 4 | — | 극위험. 특수 세력 지원 없이 접근 불가 |
| Cog 5 | — | 봉인급. 기록조차 기밀 처리 |

> 등급 명칭 및 세부 기준은 추후 확정

## 8. 세력 시스템

Cog는 이제 위험도 등급 체계 용어이므로, 세력 단위는 별도 용어로 명칭 확정 예정.
확정: **마스트(Mast)** — 세력 단위 / **앵커(Anchor)** — 각 마스트의 수장

### 세력 해금 조건

- 일정 Arc 축적 **또는** 일정 일차 경과 시 세력 시스템 해금
- 해금 이후 각 세력과 개별적으로 관계를 쌓아야 소속 인력을 고용 가능

### 세력 고용의 장점

| 구분 | 일반 고용 | 세력 고용 |
|------|-----------|----------|
| 비용 | 낮음 | 높음 (Arc 추가 필요) |
| 성능 | 기본 | 차별화된 특기 보유 |
| 스토리 | 없음 | 해당 세력 관련 이벤트 연동 |
| 리스크 | 낮음 | 세력 내부 정치에 엮일 수 있음 |

### 세력 설계 방향

- 각 세력은 단순 선악 구분 없이 고유한 논리와 역사를 보유
- 낚시꾼 위주 세력, 다이버 위주 세력, 엔지니어 집단 등 직군 특화 세력 존재
- 세력 간 관계(협력·대립·중립)가 플레이어의 선택에 영향을 줌
- 세부 세력 목록 및 설정은 세계관.md에서 관리

## 9. 부식(Rust) 시스템

- AI 몸체는 소금바람에 의해 매일 조금씩 부식된다
- 부식이 쌓이면: 능력치 저하 → 기능 장애 → 야생화
- 수리에는 Arc + 고철이 소비됨. 고품질 고철일수록 수리 효율 상승
- **설계 의도**: 자원 관리의 핵심 압박 요소

## 10. 진행 구조

- 총 Day 수: 미정 (StoryManager.js `TOTAL_DAYS = 50` 임시값)
- 하루 구성: 아침(경영) → 점심(운영) → 저녁(결산) → 다음날 아침 대화 이벤트
- Day 클리어 조건: 미정
- 영구 성장 요소: 미정
- 런(Run) 구조: 미정 (로그라이크 요소 채택 여부 포함, 현재 보류)

## 11. 개발 우선순위

```
1단계 — 뼈대
  └─ 공방(Atelier) 씬 기본 탭 레이아웃
  └─ Arc 수입 루프 (AI 배치 → Arc 수령)
  └─ 낚시꾼 보초 배치 시스템
  └─ 다이버 수집 루프

2단계 — 서사
  └─ 드레지(Dredge) 시스템 구현
  └─ 첫 번째 AI 캐릭터 스토리 씬
  └─ 로어 해금 연동

3단계 — 세계
  └─ 세력 시스템 해금 및 관계 구조
  └─ NPC 대화 시스템
  └─ 부식 시스템

4단계 — 확장
  └─ 세력 소속 전문 인력 고용
  └─ 고철 품질 등급
  └─ 추가 AI 캐릭터
```
