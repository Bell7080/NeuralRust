## 2. 장르 정의

**스토리 중심 관리 시뮬레이션 + 디펜스**

- 로보토미 코퍼레이션을 레퍼런스로 삼되, 게임성 비중은 낮추고 서사 비중을 높인다
- 직접 조작하는 액션 요소는 최소화. 플레이어는 **배치와 판단**으로 게임을 진행한다
- 매일 반복되는 운영 루틴이 플레이어를 이 세계에 살게 만들고, 그 루틴 사이에 스토리가 끼어든다
- 로그라이크 요소는 현 단계에서 보류

## 3. 핵심 루프

```
[ 아침 — 경영 페이즈 ]
  스토리 씬 / NPC 대화
  ↓
  인원 배치 · 채용 · 드레지(Dredge) 편성
  └─ AI 전류 채굴 라인 배치
  └─ 낚시꾼 보초 배치
  └─ 다이버 잠수 목표 설정
  └─ 엔지니어 외주 협상 (필요 시)

[ 점심 — 운영 페이즈 ]
  배치된 인력으로 발생한 이벤트 처리
  └─ 괴수 방어 (낚시꾼 디펜스)
  └─ 잠수 결과 수령 (고철 · 로어 · 기타)
  └─ AI 채굴 Arc 수령

[ 저녁 — 결산 페이즈 ]
  Arc · 자원 정산
  스토리 씬 / 캐릭터 대화
  └─ AI 드레지 대상 스토리 진행 가능

[ 다음날 아침 ]
  전날 이벤트 후일담 / 새 대화 이벤트
  → 아침 경영 페이즈로 순환
```

## 4. 자원 구조

### 4-1. Arc (아크) — 핵심 화폐

- 바다에 매립된 톱니바퀴에서 흘러나오는 전선의 전류를 정제한 에너지 단위
- 거대한 톱니바퀴 선박을 바다 위에 띄우는 데 필수적인 동력원이기에, 자연스럽게 이 세계의 화폐로 정착함
- AI를 전류 채굴 라인에 배치하면 매일 일정량의 Arc를 수입으로 얻는다
- 모든 고용 · 외주 · 장비 구매는 Arc로 결제

| 지출 항목 | 내용 |
|-----------|------|
| 낚시꾼 고용 | 일당 Arc 지급 |
| 다이버 고용 | 잠수 의뢰 Arc 지급 |
| 엔지니어 외주 | 하드웨어 제작 의뢰 Arc + 고철 |
| 몸체 수리 | 부식된 AI 하드웨어 유지보수 |
| Cog 소속 인력 고용 | 일반 고용보다 높은 Arc 필요 |

### 4-2. 고철 (Scrap)

- 다이버가 심해에서 수집하는 붕괴 이전 문명의 금속 잔해
- 엔지니어에게 Arc와 함께 지급해 AI 하드웨어(몸체)를 제작하는 핵심 재료
- 고철의 시대와 품질에 따라 제작되는 몸체 성능이 달라짐
- 소금바다 증기에 의해 몸체가 서서히 부식(Rust)되므로 수리용으로도 소비됨

### 4-3. 기타 수집물

- 다이버가 고철과 함께 건져올리는 다양한 물품
- 로어(기록물), 출처 불명 교역품, Cog 해금 트리거가 되는 특수 유물 등
- 각 물품의 활용 방식은 추후 설계

## 5. 직군 구조

### 5-1. AI (드레지 대상)

| 항목 | 내용 |
|------|------|
| 핵심 역할 | Arc 채굴. 전선에서 전류를 획득해 Arc를 생산 |
| 수익 구조 | 배치된 AI 수 × 개체 성능 = 일일 Arc 수입 |
| 스토리 역할 | 드레지(Dredge)를 통해 탄생한 각 AI가 독립적인 스토리를 진행하는 핵심 요소 |
| 리스크 | 몸체 부식. 방치 시 능력치 저하 → 기능 장애 → 야생화 |
| 획득 방법 | 엔지니어에게 몸체 제작 의뢰 → 드레지(Dredge) 수행 → AI 탑재 완료 |

**드레지(Dredge)**: 소프트웨어 상태로 전선을 부유하는 AI를 고철 몸체에 불러오는 행위.
심연에서 무언가를 건져올린다는 의미를 담는다.

### 5-2. 낚시꾼 (Fisher)

| 항목 | 내용 |
|------|------|
| 역할 | 톱니바퀴 선박의 외곽을 지키는 보초. 괴수 및 외부 위협으로부터 방어 |
| 게임적 기능 | 디펜스 라인 구성. 배치와 장비로 방어력 결정 |
| 고용 구조 | 초기: 일반 인물 개인 고용 / 이후: Cog 소속 낚시꾼 단체 계약 가능 |
| 리스크 | 부상 / 사망 / 정신적 소모 가능성 |

### 5-3. 다이버 (Diver)

| 항목 | 내용 |
|------|------|
| 역할 | 심해로 내려가 고철 · 로어 · 기타 물품 수집 |
| 게임적 기능 | 고철 공급 (AI 하드웨어 제작 재료) + 로어 해금 + 특수 이벤트 트리거 |
| 고용 구조 | 초기: 일반 잠수사 개인 고용 / 이후: Cog 소속 잠수사 집단 외주 가능 |
| 리스크 | 잠수 깊이에 따라 위험도 상승. 수집물 품질도 달라진다 |

### 5-4. 엔지니어 (Engineer)

| 항목 | 내용 |
|------|------|
| 역할 | AI 하드웨어(몸체) 제작 전문가 |
| 고용 구조 | 항상 외주. Arc + 고철을 지급하고 협상으로 제작 의뢰 |
| 발전 구조 | 초기: 일반 엔지니어 / 이후: Cog 소속 엔지니어 집단에 접근 가능 |
| 특이점 | Cog마다 엔지니어 작풍이 달라 몸체 특성이 차별화됨 |

## 6. Cog (코그) 세력 시스템

Cog는 이 세계의 파벌·집단을 의미하는 단위. 톱니바퀴(Cog)의 이빨 하나처럼 각 세력은 이 세계를 굴리는 맞물린 구성 요소다.

### 6-1. 해금 조건

- 일정 Arc 축적 **또는** 일정 일차 경과 시 Cog 시스템 해금
- 해금 이후 각 Cog 세력과 개별적으로 관계를 쌓아야 소속 인력을 고용 가능

### 6-2. Cog 고용의 장점

| 구분 | 일반 고용 | Cog 고용 |
|------|-----------|----------|
| 비용 | 낮음 | 높음 (Arc 추가 필요) |
| 성능 | 기본 | 차별화된 특기 보유 |
| 스토리 | 없음 | 해당 Cog 관련 이벤트 연동 |
| 리스크 | 낮음 | Cog 내부 정치에 엮일 수 있음 |

### 6-3. Cog 설계 방향

- 각 Cog는 단순 선악 구분 없이 고유한 논리와 역사를 보유
- 낚시꾼 위주 Cog, 다이버 위주 Cog, 엔지니어 집단 Cog 등 직군 특화 세력 존재
- Cog 간 관계(협력·대립·중립)가 플레이어의 선택에 영향을 줌
- 세부 Cog 목록 및 설정은 세계관.md에서 관리

## 7. 부식(Rust) 시스템

- AI 몸체는 소금바람에 의해 매일 조금씩 부식된다
- 부식이 쌓이면: 능력치 저하 → 기능 장애 → 야생화
- 수리에는 Arc + 고철이 소비됨. 고품질 고철일수록 수리 효율 상승
- **설계 의도**: 자원 관리의 핵심 압박 요소. Arc를 충분히 벌고, 고철을 충분히 확보하고, AI를 충분히 수리한다는 긴장감

## 9. 개발 우선순위

```
1단계 — 뼈대
  └─ Arc 수입 루프 (AI 배치 → Arc 수령)
  └─ 낚시꾼 보초 배치 시스템
  └─ 다이버 수집 루프

2단계 — 서사
  └─ 드레지(Dredge) 시스템 구현
  └─ 첫 번째 AI 캐릭터 스토리 씬
  └─ 로어 해금 연동

3단계 — 세계
  └─ Cog 시스템 해금 및 관계 구조
  └─ NPC 대화 시스템
  └─ 부식 시스템

4단계 — 확장
  └─ Cog 소속 전문 인력 고용
  └─ 고철 품질 등급
  └─ 추가 AI 캐릭터
```
